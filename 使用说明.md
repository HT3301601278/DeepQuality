# DeepQuality 使用说明

## 1. 项目用途

DeepQuality 用于工业过程软测量建模：输入连续采集的过程变量，预测难以在线实时测量的质量变量。

项目默认使用 `data/raw/Debutanizer_Data.csv`，该数据集包含 2394 条样本、7 个过程变量 `u1` 到 `u7`、1 个质量变量 `y`。

项目还提供 `data/raw/SRU_Data.csv`，该数据集包含 10081 条样本、5 个过程变量 `u1` 到 `u5`、2 个质量变量 `y1`、`y2`。

项目主要包含两类模型：

1. `S-DDAE`：监督动态降噪自编码器，作为基线软测量模型。
2. `SS-DDFAE`：半监督动态深度融合自编码器，用少量有标签样本和大量无标签样本训练改进模型。

一次完整展示建议覆盖以下内容：

1. 数据格式。
2. 配置文件。
3. S-DDAE 基线训练。
4. SS-DDFAE 半监督训练。
5. 后处理参数搜索。
6. checkpoint 评估。
7. 在线推理模拟。
8. 指标表格和预测图展示。

## 4. 数据说明

默认数据文件：

```text
data/raw/Debutanizer_Data.csv
```

表头如下：

```text
u1,u2,u3,u4,u5,u6,u7,y
```

其中：

| 列 | 含义 |
|---|---|
| `u1` 到 `u7` | 过程变量 |
| `y` | 质量变量，也是默认预测目标 |

SRU 数据文件：

```text
data/raw/SRU_Data.csv
```

表头如下：

```text
u1,u2,u3,u4,u5,y1,y2
```

如果使用 SRU 数据，需要在配置中把 `data.path` 改为 `data/raw/SRU_Data.csv`，并把 `data.target_column` 改成 `y1` 或 `y2`。

自定义数据必须满足：

1. 使用 CSV 格式。
2. 第一行必须是表头。
3. 目标列名必须写在配置项 `data.target_column` 中。
4. 除目标列之外的所有列都会作为过程变量。
5. 所有数据列都应为数值。
6. 空字符串会被读取为缺失值。

数据处理流程：

1. 读取 CSV。
2. 按时间顺序划分训练集、验证集、测试集。
3. 删除连续缺失长度大于 `5` 的片段。
4. 如果任一列缺失率大于等于 `0.01`，程序报错。
5. 对剩余缺失值按列线性插值。
6. 只用训练集统计量做标准化。
7. 对输入和目标执行 `3σ` 裁剪。
8. 构造动态窗口。
9. 计算过程变量相关性权重。
10. 根据 `label_ratio` 构造有标签样本掩码。

默认数据划分：

| 集合 | 比例 |
|---|---:|
| 训练集 | `0.70` |
| 验证集 | `0.15` |
| 测试集 | `0.15` |

## 5. 配置文件

配置文件采用 YAML。

加载配置时，程序会先读取：

```text
configs/base.yaml
```

再读取命令中指定的模型配置，并覆盖同名字段。

基础配置：

```yaml
seed: 3
device: auto

data:
  path: data/raw/Debutanizer_Data.csv
  target_column: y
  train_ratio: 0.70
  val_ratio: 0.15
  window_size: 40
  quality_delay: 12
  label_ratios: [0.2, 0.3, 0.5, 1.0]
  scales: []

paths:
  logs_dir: artifacts/logs
  figures_dir: artifacts/figures
  checkpoints_dir: artifacts/checkpoints
  tables_dir: artifacts/tables
```

关键字段：

| 字段 | 含义 |
|---|---|
| `seed` | 随机种子 |
| `device` | 训练设备，`auto` 表示自动使用 CUDA 或 CPU |
| `data.path` | CSV 数据路径 |
| `data.target_column` | 目标变量列名 |
| `data.train_ratio` | 训练集比例 |
| `data.val_ratio` | 验证集比例 |
| `data.window_size` | 单尺度动态窗口长度 |
| `data.quality_delay` | 质量变量延迟步数 |
| `data.label_ratios` | 生成嵌套标签掩码时使用的标签比例集合 |
| `data.scales` | 多尺度窗口配置 |
| `paths.checkpoints_dir` | checkpoint 输出目录 |
| `paths.tables_dir` | 指标和预测 CSV 输出目录 |
| `paths.figures_dir` | 图像输出目录 |

单尺度 S-DDAE 配置：

```text
configs/sddae_single_scale.yaml
```

用途：训练 SS-DDFAE 前置基线，也用于常规单尺度基线对比。

多尺度 S-DDAE 配置：

```text
configs/sddae.yaml
```

默认多尺度：

```yaml
scales:
  - [12, 1]
  - [24, 2]
  - [48, 4]
```

含义：

| 配置 | 含义 |
|---|---|
| `[12, 1]` | 窗口长度 12，采样步长 1 |
| `[24, 2]` | 窗口长度 24，采样步长 2 |
| `[48, 4]` | 窗口长度 48，采样步长 4 |

SS-DDFAE 配置：

```text
configs/ss_ddfae.yaml
```

用途：训练半监督动态深度融合自编码器。

SS-DDFAE 当前只支持单尺度输入。训练 SS-DDFAE 时，`--baseline-checkpoint` 必须传入由 `configs/sddae_single_scale.yaml` 训练出的单尺度 S-DDAE checkpoint。

## 6. 完整展示流程

以下流程从零开始运行，最终得到 checkpoint、指标表格、预测结果 CSV、损失曲线、预测曲线、散点图、残差图和在线推理延迟结果。

### 6.1 训练 S-DDAE 基线模型

运行：

```bash
python -m deep_quality.cli.train_sddae \
  --config configs/sddae_single_scale.yaml \
  --label-ratio 0.2 \
  --window-size 40 \
  --quality-delay 12 \
  --latent-dim 32 \
  --output-name sddae_r_L40_d12_z32_r0.2
```

参数含义：

| 参数 | 含义 |
|---|---|
| `--config` | 使用单尺度 S-DDAE 配置 |
| `--label-ratio 0.2` | 只使用 20% 训练窗口作为有标签样本 |
| `--window-size 40` | 每个样本使用 40 个历史时刻构造动态窗口 |
| `--quality-delay 12` | 用当前窗口预测 12 步后的质量变量 |
| `--latent-dim 32` | 编码器隐变量维度为 32 |
| `--output-name` | 输出文件名前缀 |

训练完成后会生成：

```text
artifacts/checkpoints/sddae_r_L40_d12_z32_r0.2.pt
artifacts/tables/sddae_r_L40_d12_z32_r0.2_metrics.json
artifacts/tables/sddae_r_L40_d12_z32_r0.2_metrics.csv
artifacts/tables/sddae_r_L40_d12_z32_r0.2_predictions.csv
artifacts/figures/sddae_r_L40_d12_z32_r0.2_loss.png
artifacts/figures/sddae_r_L40_d12_z32_r0.2_prediction.png
artifacts/figures/sddae_r_L40_d12_z32_r0.2_scatter.png
artifacts/figures/sddae_r_L40_d12_z32_r0.2_residuals.png
```

其中：

| 文件 | 内容 |
|---|---|
| `.pt` | 模型 checkpoint |
| `_metrics.json` | 测试集 RMSE、MAE、R2 等指标 |
| `_metrics.csv` | CSV 格式指标 |
| `_predictions.csv` | 测试集真实值和预测值 |
| `_loss.png` | 训练和验证损失曲线 |
| `_prediction.png` | 真实值和预测值曲线 |
| `_scatter.png` | 真实值-预测值散点图 |
| `_residuals.png` | 残差曲线 |

运行：

```bash
python -m deep_quality.cli.train_ss_ddfae \
  --config configs/ss_ddfae.yaml \
  --baseline-checkpoint artifacts/checkpoints/sddae_r_L40_d12_z32_r0.2.pt \
  --label-ratio 0.2 \
  --window-size 40 \
  --quality-delay 12 \
  --latent-dim 32 \
  --output-name ss_ddfae_L40_d12_z32_r0.2
```

参数含义：

| 参数 | 含义 |
|---|---|
| `--config` | 使用 SS-DDFAE 配置 |
| `--baseline-checkpoint` | 读取 S-DDAE checkpoint 并迁移兼容参数 |
| `--label-ratio 0.2` | 半监督训练中 20% 样本有标签，其余训练样本作为无标签样本 |
| `--window-size 40` | 与基线模型保持一致 |
| `--quality-delay 12` | 与基线模型保持一致 |
| `--latent-dim 32` | 与基线模型保持一致 |
| `--output-name` | 输出文件名前缀 |

```text
artifacts/tables/sddae_r_L40_d12_z32_r0.2_metrics.json
artifacts/tables/ss_ddfae_L40_d12_z32_r0.2_metrics.json
```

指标字段：

| 字段 | 含义 |
|---|---|
| `model` | 模型名称 |
| `label_ratio` | 有标签样本比例 |
| `window_size` | 动态窗口长度 |

| `rmse` | 均方根误差，越低越好 |
| `mae` | 平均绝对误差，越低越好 |
| `r2` | 决定系数，越高越好 |

### 6.3 搜索后处理方法

训练完成后，先对 checkpoint 搜索后处理方法：

```bash
python -m deep_quality.cli.tune_postprocess \
  --checkpoint artifacts/checkpoints/ss_ddfae_L40_d12_z32_r0.2.pt
```

该命令会在验证集上比较以下后处理方法：

| 方法 | 含义 |
|---|---|
| `raw` | 不做校正 |
| `ema` | 指数滑动平均 |
| `ar` | AR(1) 残差校正 |
| `ema+ar` | 先 EMA 平滑，再做 AR(1) 残差校正 |

输出文件：

```text
artifacts/tables/ss_ddfae_L40_d12_z32_r0.2_postprocess.csv
artifacts/tables/ss_ddfae_L40_d12_z32_r0.2_postprocess.json
```

其中：

| 文件 | 内容 |
|---|---|
| `_postprocess.csv` | 所有候选后处理方法的验证集指标 |
| `_postprocess.json` | 最优后处理方法、参数和指标摘要 |

后续评估和在线推理会读取 `_postprocess.json`。如果没有先执行该步骤，评估或在线推理会报错：

```text
未找到后处理摘要
```

### 6.4 评估 checkpoint

运行：

```bash
python -m deep_quality.cli.evaluate_checkpoint \
  --checkpoint artifacts/checkpoints/ss_ddfae_L40_d12_z32_r0.2.pt
```

该命令会：

1. 加载 checkpoint。
2. 重新按 checkpoint 中保存的配置处理数据。
3. 在训练集、验证集、测试集上生成原始预测。
4. 读取后处理摘要。
5. 对测试集预测结果应用后处理。
6. 输出后处理后的 RMSE、MAE、R2。

输出文件：

```text
artifacts/tables/ss_ddfae_L40_d12_z32_r0.2_eval_metrics.json
artifacts/tables/ss_ddfae_L40_d12_z32_r0.2_eval_metrics.csv
artifacts/tables/ss_ddfae_L40_d12_z32_r0.2_eval_predictions.csv
artifacts/figures/ss_ddfae_L40_d12_z32_r0.2_eval_prediction.png
artifacts/figures/ss_ddfae_L40_d12_z32_r0.2_eval_scatter.png
artifacts/figures/ss_ddfae_L40_d12_z32_r0.2_eval_residuals.png
```

评估指标文件额外包含：

| 字段 | 含义 |
|---|---|
| `postprocess_method` | 最终使用的后处理方法 |
| `raw_rmse` | 后处理前 RMSE |
| `raw_mae` | 后处理前 MAE |
| `raw_r2` | 后处理前 R2 |
| `rmse` | 后处理后 RMSE |
| `mae` | 后处理后 MAE |
| `r2` | 后处理后 R2 |

### 6.5 模拟在线推理

运行：

```bash
python -m deep_quality.cli.simulate_online_inference \
  --checkpoint artifacts/checkpoints/ss_ddfae_L40_d12_z32_r0.2.pt
```

该命令会逐条遍历测试集窗口，模拟在线场景下的单步预测。

每个测试样本的流程：

1. 读取当前窗口。
2. 模型前向推理。
3. 将预测值反标准化回原始量纲。
4. 按最优后处理方法做单步校正。
5. 记录单样本推理耗时。
6. 保存预测结果和平均延迟。

输出文件：

```text
artifacts/tables/ss_ddfae_L40_d12_z32_r0.2_online.csv
artifacts/tables/ss_ddfae_L40_d12_z32_r0.2_online_metrics.json
artifacts/figures/ss_ddfae_L40_d12_z32_r0.2_online_prediction.png
```

在线结果字段：

| 字段 | 含义 |
|---|---|
| `rmse` | 在线模拟测试集 RMSE |
| `mae` | 在线模拟测试集 MAE |
| `r2` | 在线模拟测试集 R2 |
| `avg_latency_ms` | 单样本平均推理耗时，单位毫秒 |
| `postprocess_method` | 在线推理使用的后处理方法 |

```bash
python -m deep_quality.cli.train_sddae
```

参数：

| 参数 | 是否必填 | 默认值 | 含义 |
|---|---|---|---|



## 8. 多尺度 S-DDAE

运行默认多尺度配置：

```bash
python -m deep_quality.cli.train_sddae \
  --config configs/sddae.yaml \
  --label-ratio 1.0 \
  --quality-delay 12 \
  --latent-dim 32 \
  --output-name sddae_r_ms12x1-24x2-48x4_d12_z32_r1
```

手动指定多尺度：

```bash
python -m deep_quality.cli.train_sddae \
  --config configs/sddae.yaml \
  --label-ratio 1.0 \
  --quality-delay 12 \
  --latent-dim 52 \
  --scales 40x1,24x2,12x4 \
  --output-name sddae_r_ms40x1-24x2-12x4_d12_z52_r1
```

多尺度参数格式：

```text
窗口长度x采样步长,窗口长度x采样步长,窗口长度x采样步长
```

例如：

```text
40x1,24x2,12x4
```

表示同时使用：

| 尺度 | 窗口长度 | 采样步长 |
|---|---:|---:|
| 短期 | 12 | 4 |
| 中期 | 24 | 2 |
| 长期 | 40 | 1 |

SS-DDFAE 不能直接使用多尺度 checkpoint 初始化。训练 SS-DDFAE 时必须使用单尺度 S-DDAE checkpoint。

## 10. 输出文件解读

### 10.1 checkpoint

路径：

```text
artifacts/checkpoints/*.pt
```

checkpoint 内部保存：

| 字段 | 含义 |
|---|---|
| `model` | 模型名称 |
| `state_dict` | PyTorch 模型参数 |
| `config` | 训练时使用的完整配置 |
| `metrics` | 训练结束时的测试集指标 |
| `baseline_checkpoint` | SS-DDFAE 使用的基线 checkpoint 路径 |

### 10.2 指标 JSON

路径：

```text
artifacts/tables/*_metrics.json
```

常见字段：

| 字段 | 含义 |
|---|---|
| `model` | `sddae_r` 或 `ss_ddfae` |
| `label_ratio` | 有标签样本比例 |
| `window_size` | 动态窗口长度 |
| `quality_delay` | 质量延迟 |
| `latent_dim` | 隐变量维度 |
| `scales` | 多尺度窗口配置，单尺度时为空 |
| `rmse` | 测试集 RMSE |
| `mae` | 测试集 MAE |
| `r2` | 测试集 R2 |

### 10.3 预测 CSV

路径：

```text
artifacts/tables/*_predictions.csv
```

字段：

| 字段 | 含义 |
|---|---|
| `index` | 测试集窗口序号 |
| `y_true` | 真实质量变量 |
| `y_pred` | 模型预测质量变量 |

在线推理 CSV 还会包含：

| 字段 | 含义 |
|---|---|
| `latency_ms` | 单样本推理耗时，单位毫秒 |

### 10.4 图像

路径：

```text
artifacts/figures/*.png
```

图像类型：

| 后缀 | 含义 |
|---|---|
| `_loss.png` | 训练损失和验证损失曲线 |
| `_prediction.png` | 测试集真实值和预测值曲线 |
| `_scatter.png` | 真实值-预测值散点图 |
| `_residuals.png` | 残差曲线 |
| `_eval_prediction.png` | 后处理评估预测曲线 |
| `_eval_scatter.png` | 后处理评估散点图 |
| `_eval_residuals.png` | 后处理评估残差图 |
| `_online_prediction.png` | 在线模拟预测曲线 |

## 13. 设备选择

配置中默认：

```yaml
device: auto
```

2. 如果没有 CUDA，训练使用 CPU。

```bash
python -m deep_quality.cli.evaluate_checkpoint \
  --checkpoint artifacts/checkpoints/ss_ddfae_L40_d12_z32_r0.2.pt \

```

使用 GPU 推理：

```bash
python -m deep_quality.cli.evaluate_checkpoint \

  --device cuda
```

## 14. 常见问题

### 14.1 `python: command not found`

创建虚拟环境前使用：

```bash
python3 -m venv .venv
```

激活虚拟环境后使用：

```bash
python -m deep_quality.cli.train_sddae
```

### 14.2 `ModuleNotFoundError: No module named 'torch'`

说明依赖未安装。执行：

```bash
python -m pip install -e .
```

### 14.3 `未找到后处理摘要`

说明还没有执行后处理搜索。先运行：

```bash
python -m deep_quality.cli.tune_postprocess \
  --checkpoint artifacts/checkpoints/ss_ddfae_L40_d12_z32_r0.2.pt
```

再运行评估或在线推理。

### 14.4 `SS-DDFAE 训练只支持单尺度基线 checkpoint`

说明传入了多尺度 S-DDAE checkpoint。重新训练单尺度基线：

```bash
python -m deep_quality.cli.train_sddae \
  --config configs/sddae_single_scale.yaml \
  --label-ratio 0.2 \
  --window-size 40 \
  --quality-delay 12 \
  --latent-dim 32 \
  --output-name sddae_r_L40_d12_z32_r0.2
```

再训练 SS-DDFAE。

### 14.5 `未找到目标列`

说明 `configs/base.yaml` 中的 `data.target_column` 与 CSV 表头不一致。检查 CSV 第一行，把 `target_column` 改成真实存在的目标列名。
